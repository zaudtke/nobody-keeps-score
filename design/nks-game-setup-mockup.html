① Fresh Start — Design Notes

  • Add player row disabled: Shown at reduced opacity with "Select a game first" label until a game card is tapped. Enforces the intended top-down flow.
  • Start button generic: Label reads "Start Game" with disabled state until both a game is selected and the minimum player count is met.

② Game Selected (Dirty Clubs) — Design Notes

  • Selected state: Felt-green border + tinted bg + filled ✓ check circle. One card at a time — tapping another deselects the current one.
  • Player count badge: Updates to "0 of 3 – 6" immediately — the range comes from the game's constraints. Muted until minimum is met.
  • CTA label updates: "Start Game" → "Start Dirty Clubs" as soon as a game is selected. No win-direction panel appears for fixed-rule games.

③ Open Scoring — High Score Wins — Design Notes

  • Card shape change on selection: When Open Scoring is selected, the card's bottom border-radius is removed (0 0 on bottom corners) so the win-direction panel attaches flush beneath it — visually one connected surface.
  • Win direction panel: Appears only for Open Scoring. Tinted felt-green bg + subtle border. Contains the toggle and the optional game name input.
  • Toggle pill: Two side-by-side buttons inside a bordered container. "High Score Wins" is active by default — felt-green fill. "Low Score Wins" is inactive/muted. Tapping switches the active state.
  • Hint text: Small line below the toggle: "↑ High score wins · host ends the game manually" — sets expectation that there's no auto-win condition.
  • Game Name input: Optional free-text field. Placeholder examples reflect the win direction — "e.g. Scrabble, Yahtzee…" for High. When filled, the Start button label uses the name: "Start Scrabble".
  • CTA hint: "3 players · ↑ high score wins" — summarises both key settings at a glance before committing.

④ Open Scoring — Low Score Wins — Design Notes

  • Amber active state: Low Score Wins uses an amber/gold tint rather than felt-green — visually distinct from the default, signals a different win condition at a glance. Consistent with the amber = low-wins convention used on the scoreboard win-direction chip.
  • Hint text updates: "↓ Low score wins · host ends the game manually" — matches the new direction.
  • Game Name placeholder updates: When Low wins is active, placeholder suggests "e.g. Golf, Mini-golf…" — contextually appropriate examples.
  • Start button uses game name: When the host has typed "Golf", the CTA reads "✏️ Start Golf" instead of "✏️ Start Open Scoring" — personal and specific.
  • CTA hint: "3 players · ↓ low score wins" — direction symbol and label both update.

⑤ Ready — Dirty Clubs — Design Notes

  • Game list collapses: Once a game is selected and the host moves to adding players, the game type list compresses to just the selected card — conserves vertical space on iPhone SE.
  • Player count badge turns green: "4 of 3 – 6" in felt tint once minimum is met.
  • Start button activates: Full gradient + shadow. Label: "♣ Start Dirty Clubs". CTA hint: rule summary "first to 15 points wins".

⑥ Returning Players — Design Notes

  • Context banner: "🔁 Players carried over · From previous Dirty Clubs game" at the top — warm, not alarming. Explains the pre-populated state.
  • App bar sub-label: "Same night · adjust if needed" — contextual signal this is a continuation.
  • Full game list shown: Host may want to switch game type for the next round — all four options visible and tappable. If they switch to Open Scoring, the win direction panel appears as normal.
  • Start immediately active: Carried-over player count satisfies the minimum — host can tap straight through without any changes.
