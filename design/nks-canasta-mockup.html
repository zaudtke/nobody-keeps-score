Scoreboard — Design Notes

  • Meld badge: Colored pill in the player sub-row. Meld 50 = felt green (low stakes). Meld 90 = amber (mid). Meld 120 = ruby (high). No min. = muted ink (negative score). Updates immediately after each round.
  • Score thresholds: <0 → No min. · 0–1499 → Meld 50 · 1500–2999 → Meld 90 · ≥3000 → Meld 120. Negative rounds can drop a player to a lower tier — badge updates accordingly.
  • Near Win highlight: Gold left-blush + "Near Win" badge appears when a player crosses 4,000. Mirrors the Dirty Clubs forced-play treatment.
  • Negative score: Score numeral turns ruby-red. Meld badge shows "No min." in muted ink. Players are still in the game.
  • Round history: Compact table below standings. Positive deltas in felt-green, negative in ruby. Round number in left column.
  • Score entry sheet: Scoreboard uses Option C (stacked rows) as the default — compact, clear, fast. Tap any player row to open.

Entry Option A — Two Separate Fields

  • Both fields always visible. Base (amber) and Score (green) are always on screen. No hidden state, no discovery required.
  • Color-coded labels and borders. Base field uses amber tones; Score uses felt-green. Instant visual distinction even at a glance.
  • Inline hints. "Adjustment from previous scoring · default 0" and "Points earned this round" reduce the need for players to remember the rules.
  • Live math bar. prev + base + score = result shown at the top. Updates as you type. Result turns ruby if negative, gold if crossing 5,000.
  • Tradeoff: Most explicit and safest for new players. Slightly more vertical space than the other options.

Entry Option B — Score Primary + Base Expander

  • Score is the hero. Large centered input — dominant target on a small screen. Fast for the 95% of rounds where Base = 0.
  • Base hidden by default. A subtle "+ Add base adjustment (usually 0)" toggle reveals the amber base field only when needed. Zero visual noise on typical rounds.
  • Chevron rotates on expand. Clear affordance — the row opens smoothly, the field slides in. Collapses again if the toggle is tapped.
  • Live math bar. Always shows the full equation regardless of whether Base is expanded — base just shows 0 when collapsed.
  • Tradeoff: Cleanest common-case UX. Players who need Base must know to look for the toggle — slight discoverability cost.

Entry Option C — Stacked Inline Rows

  • Both fields always visible, compact. Two rows stacked vertically — label chip on the left, number input on the right. Familiar iOS form feel.
  • Focus state activates the row. Tapping a row highlights the full row border and label background in the field's color (amber for Base, green for Score).
  • Compact height. The two rows together are shorter than Option A's two labeled blocks. Leaves more room for the live math and confirm button without scrolling.
  • Live math bar. Same prev + base + score = result equation. Updates on every keystroke.
  • Tradeoff: Most space-efficient. The row labels (Base / Score) are smaller than Option A — may need a moment's reading for new users. Ideal for regular players who know the system.

Phase 1 — Batch Base Entry

  • "Score Round" button appears at the top of the scoreboard scrollable area. Tapping it opens this sheet — no per-player row tap needed to start a round.
  • All players listed at once. Avatar, name, and current score shown for context. Each player has a compact amber Base input on the right. Any order — fill in whoever is ready first.
  • Base defaults to 0. Placeholder shows "0". If nothing is entered, 0 is used when bases are locked. No required entry — keeps the flow fast when base is zero for everyone.
  • "Lock Bases & Start Scores" button confirms all base values and closes the sheet. Phase 2 begins immediately — the scoreboard returns and the host taps each player as they finish counting cards.

Phase 2 — Score Entry (per player, any order)

  • Same scoreboard, same tap-to-open flow. No visual change to player rows — the host taps whoever is ready first. No queue, no waiting.
  • Score-only sheet. The Base field is gone — base was already locked in Phase 1. Only the Score input appears, reducing entry to a single field per player.
  • Locked base in the math bar. The base value from Phase 1 is shown in amber with a small "base" chip so it's clear it's already confirmed. The full equation prev + base + score = result still updates live as Score is typed.
  • Confirm closes the sheet. Running total and meld badge update immediately. Repeat for remaining players until the round is complete.

Game Over Screen — Design Notes

  • Multiple finishers handled. When more than one player crosses 5,000 in the same round, the highest score wins. The hero note calls this out explicitly — "Jim finished at 4,950 — Sandra wins."
  • Tie note. In a true tie, the note would read "Tied — break at the table." No app-level tiebreaker needed.
  • Negative finisher. Patty's final score shows in ruby red with a "Finished negative" detail line — acknowledged without drama.
  • Placement numbers. Gold 1st, silver-ink 2nd, amber 3rd, muted for lower places. Matches NKS trophy/gold token conventions.
  • Actions: "New Game" routes back to Game Setup. "End Night" archives the session.
