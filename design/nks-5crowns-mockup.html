Scoreboard — Design Notes

  • Round strip is always visible. The large round label (3–K) and pip track sit directly below the app bar. The host can see the current round, card count, wild card, and remaining rounds at a glance without scrolling.
  • Pip track uses round labels, not numbers. Pips are labelled 3, 4, 5 … 10, J, Q, K. Done pips are muted green, the current round is gold and glow-highlighted, future rounds are dark ghost.
  • Complete Round button lives in the strip. It shows the remaining player count when waiting ("2 remaining"), activates once all players are scored ("Complete Round →"), and turns gold for the final round ("End Game →").
  • Lowest score leads. Players are sorted ascending. The leading player gets a gold left-blush highlight and a "Leading" badge — mirroring the near-win treatment in Canasta but inverted for low-score games.
  • Per-player round status. Each row shows either "awaiting round score" or a ✓ check with the delta for this round (e.g. ✓ +31). This lets the host know who still needs to be entered without opening any sheet.

Score Entry Sheet — Design Notes

  • Single field only. 5 Crowns scoring is straightforward — one number per player per round. No Base field is needed here (unlike Canasta). The sheet is intentionally minimal.
  • Round context chip in header. The gold chip below the player name shows the current round label and wild card (e.g. "Round J · Jacks wild") so the host knows exactly which round they're entering without needing to check the scoreboard.
  • Live math bar. prev + this round = new total. Updates on every keystroke. New total turns ruby if negative (edge-case only). Prev and total use Fraunces for visual weight; the operator and equals are muted ink.
  • Zero is valid. A player who goes out scores 0. Entering 0 enables the Confirm button. The hint text "Enter 0 if player went out" reminds the host of this.
  • Scoreboard dimmed above. The scoreboard remains partially visible above the sheet so all players can follow standings while the host enters scores.

All Scores In — Design Notes

  • "Complete Round →" activates. Once every player has a confirmed score for the current round, the button in the round strip switches from disabled ("3 remaining") to active green ("Complete Round →"). Host-only action.
  • All rows show ✓ check. Every player row shows their delta for this round with a green check — giving the host a quick visual confirmation before advancing.
  • Went out shown explicitly. A delta of +0 renders as "✓ +0 · went out" rather than just "+0" — the secondary label makes the context clear.
  • Last round label changes. When Round Q is complete and all scores are in, the button reads "End Game →" in gold instead of green, signalling the final action.

Game Over Screen — Design Notes

  • Crown emoji, not trophy. 5 Crowns uses a 👑 crown to match the game's theme, distinguishing it from the 🏆 trophy used in Canasta.
  • Winner = lowest score. The hero note calls this out explicitly: "Final score: 298 · lowest of the night" and the footer line reads "Lowest score wins · well played." so there's no ambiguity.
  • Placement numbers match Canasta pattern. Gold 1st, silver-ink 2nd, amber 3rd — consistent across all NKS game types.
  • Round K delta shown. Each row shows the player's delta on the final round as context, including "went out" callout for zero scores.
  • Actions: End Night / New Game. Consistent with Canasta. "New Game" routes to Game Setup; "End Night" archives the session.
