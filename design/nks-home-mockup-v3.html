① Unauthenticated — Design Notes

  • Single action: The entire screen leads to one outcome — Sign in with Google. Everything above it is context, not navigation.
  • App description: One sentence, centred, muted. Sets the purpose without over-explaining. "Fast to start, simple to track, satisfying to settle."
  • Games grid: 2×2 — all four games equal size. Informational only, not tappable here. Icon + name + genre descriptor gives just enough context without detailing rules.
  • Google Sign-In button: Outlined style with real Google G logo SVG. Not a filled primary button — this is a personal app, not a conversion funnel. Hint text "Host access only · Private use" signals the intent without being alarming.
  • Auth flow: On tap → Firebase Google OAuth popup → on success, app queries authorisedUsers/{uid} in Firestore → if found, route to / (authenticated state) → if not found, sign out Firebase user and route to /unauthorized.

② Host Signed In — Design Notes

  • Host avatar in app bar: First initial of the Google display name, gold ring. Replaces the theme toggle — signals authenticated state persistently without a text label.
  • Personal greeting: Time-aware (morning / afternoon / evening) + first name from Google profile. Warm and personal — this is always the same one user.
  • Start Game Night: The singular CTA. Large Fraunces button with green gradient and a ghosted ♣ watermark. Routes to Game Setup. Nothing else competes for attention.
  • Games pill row: Compact secondary reference — all four games at a glance before entering setup. Not a navigation element.
  • Sign out: Small, muted, below the fold on iPhone SE. Present but unobtrusive — won't be tapped accidentally.
  • No session history: Deliberately omitted for v1. Firestore stores session data but there is no archive UI in this release.

③ Unauthorised · /unauthorized — Design Notes

  • Separate route: Alice lands at /unauthorized — not the home screen. The app bar is minimal (no host controls, no avatar). She knows she's in the wrong place.
  • Tone — friendly and vague: "This app is for private use" — no technical detail, no "you have been denied", no mention of allowlists or Firestore. Warm but final.
  • Card suit icon: 🃏 rather than a lock or error symbol — maintains the app's personality even in an error state. Not alarming.
  • Try a different account: The only action available. Outlined button with Google logo — triggers a fresh Google sign-in popup, which re-runs the full auth + Firestore check flow. Firebase user from the failed attempt is signed out before this screen renders.
  • NKS signature: "Nobody's Keeping Score" in small italic Fraunces at the bottom — a quiet brand echo. Reinforces that this is an intentional, designed app, not a broken one.
  • Auth flow on this screen: On "Try a different account" tap → signOut() then re-trigger Google OAuth → same authorisedUsers check → authorised routes to /, unauthorised loops back to /unauthorized.
