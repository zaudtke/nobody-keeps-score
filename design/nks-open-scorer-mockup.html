Scoreboard (High Score Wins) — Design Notes

  • Round counter in the strip. "Round N" sits in the game strip alongside the win direction chip. A plain incrementing number — no max, no rules attached. Updates when the host taps Complete Round.
  • Complete Round button — same pattern as 5 Crowns. Disabled ("N remaining") until every player has a confirmed score for the current round. Activates to "Complete Round →" in felt green once all scores are in. Host-only action.
  • End Game always active. Sits below Complete Round in the strip. The host can end the game at any time regardless of whether the current round is fully scored.
  • Win direction chip. "↑ High wins" in felt green or "↓ Low wins" in amber — always visible, set once at Game Setup.
  • Per-player round status. Each row shows "✓ scored" or "awaiting round score" — identical treatment to 5 Crowns. The High screen shows a mid-round state (2 scored, 2 waiting). The Low screen shows all scored with Complete Round active.
  • Sort order follows win direction. High wins = descending. Low wins = ascending. Section label adapts accordingly.

Scoreboard (Low Score Wins, All Scored) — Design Notes

  • Complete Round active. All four players show ✓ scored, so the Complete Round button is green and tappable. This is the state the host sees just before advancing to the next round.
  • Amber chip for Low wins. "↓ Low wins" in amber — visually distinct from the felt green High wins chip so the direction is immediately clear.
  • Leader = lowest total. Jim leads with 142. Gold blush and "Leading" badge on the top row as always, regardless of direction.

Score Entry Sheet — Design Notes

  • Negative turns are first-class. The Open Scorer fully supports negative turn values for penalties and corrections. When the value goes negative, the input border turns ruby, the label chip turns ruby, the math bar turn value turns ruby, and the hint text updates to "Negative — penalty or correction".
  • Simple header — no round chip. Just the player name and current total. The round context lives in the always-visible game strip above, so there's no need to repeat it in the sheet.
  • Live math bar. prev + turn = new total updates on every keystroke. New total turns ruby if it goes negative.
  • Confirm enables on any value. Positive, zero, and negative all activate the Confirm button once the field has content. After confirming, the player row shows ✓ scored and counts toward the Complete Round threshold.

Game Over Screen — Design Notes

  • Win direction reflected in hero text. High wins: "highest of the night". Low wins: "lowest of the night". Note line adapts to match.
  • Triggered by End Game, not round completion. The host can end mid-round if needed — the Game Over screen shows whatever totals exist at that moment.
  • Negative final scores shown in ruby. Edge case, but handled gracefully.
  • Actions consistent with all NKS games. "End Night" archives the session; "New Game" routes to Game Setup.
