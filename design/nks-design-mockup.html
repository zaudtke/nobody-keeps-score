① Home Screen — Design Notes

  • Two states: Visitor (join only) and Host (create + join). The admin "Host access" link is intentionally subtle — small, muted text below the join form.
  • Room code input: Fraunces display font at large size with wide letter-spacing. 4 digits only, centered. Feels tactile, like a safe combination.
  • Card suits in hero: ♠ ♥ ♦ ♣ as a decorative motif — repeats throughout the app for thematic consistency.
  • Theme toggle: Top-right of the app bar. Persists across sessions via localStorage.
  • Safe zone: Content sits well above the 375px width — no overflow issues on iPhone SE.

② Lobby Screen — Design Notes

  • Room code hero: Large digit blocks in felt-600 (light) / felt-800 (dark) — prominent and easy to share verbally or visually.
  • Player rows: Avatar initial + name + status badge. Three badge states: host (felt-600 solid), claimed (gold tint), claim (felt tint, tappable).
  • Host star indicator: The top-right icon-btn gets a gold tint when the current user is the host — a persistent but non-intrusive signal.
  • Tab bar: 3 tabs: Lobby / Scores / Archive. Felt-600 active state. Scores tab becomes active once a game is running.
  • CTA for host: "Start First Game" button only visible in host view — not shown to viewers.

③ Game Setup Screen — Design Notes

  • Game type cards: Selected state uses felt-600 border + subtle felt-50/felt-950 background tint. Rounded check circle fills on selection.
  • Player toggle: Tap rows to include/exclude players from this game. Checkmark (felt-500) vs empty circle (ink-600).
  • CTA label updates: "Start Rummy" / "Start Whist" / "Start Open Scorer" — the button reflects the selected type.
  • Host-only route: This screen is unreachable for non-hosts (Firestore rules enforce this server-side too).

④ Scoreboard Screen — Design Notes

  • Winner banner: Gold gradient strip at top — "Currently Winning" updates in real-time via Firestore. Trophy emoji + leading player + total.
  • Score table: Per-round columns (R1, R2, R3…) with Fraunces numerals. Medal emoji for rank 1/2/3. Compact — fits 4 players on a 375px screen.
  • Round entry (host only): Stepper widget (−/value/+) — thumb-friendly, large tap target. Each player gets their own row. "Save Round" writes to Firestore.
  • Claimed player entry: A claimed player sees their own stepper row enabled; other players' rows are read-only.
  • End Night / New Game: Ruby-bordered "End Night" uses a destructive color — intentionally alarming. "New Game" routes to Game Setup.

⑤ Archive Screen — Design Notes

  • Grouped by month: Section headers act as temporal dividers — Fraunces display treatment for month labels.
  • Archive card: Room code (large, felt-700/400) + date/players/game count + game type pills + winner callout aligned right.
  • Game type pills: Small felt-tinted badges — easy to scan which games were played each night.
  • Winner display: Gold-tinted label + player name — celebrates the evening's champion persistently.
  • Summary count: Footer hint shows total sessions + games played — satisfying for the nostalgic tone.
